# Pizza-Sales-Performance-Strategic-Analysis-Report
Kaggle Link: (https://www.kaggle.com/datasets/mysarahmadbhat/pizza-place-sales)
## 📌 Project Overview
This project performs a comprehensive Data Analysis (EDA) on a pizza restaurant's sales dataset. The goal is to uncover actionable insights regarding customer behavior, peak sales periods, and menu performance to help the business increase revenue and optimize operations.

## 🛠️ Tech Stack
* SQL Analysis Snippets
   In a real-world production environment, the data would be queried from a database.
* *Language:* Python
* *Libraries:* * *Pandas:* For data manipulation and cleaning.
* *Matplotlib & Seaborn:* For static data visualization.
* *NumPy:* For numerical operations.

## 📊 Key Features & Analysis
* *Sales Trends:* Analysis of sales by Hour, Day, and Month.
* *Customer Preferences:* Distribution of pizza categories and sizes.
* *Revenue Analysis:* Calculation of total revenue per quantity and pizza type.
* *Operational Insights:* Identification of peak hours (Lunch rush) and slow periods.
* *Advanced Visualization:* Custom bar charts highlighting the Best and Worst performing categories using conditional coloring.

## 📈 Key Findings
1.  *Top Seller:* The "Classic Deluxe Pizza" is the most popular item.
2.  *Peak Time:* 12:00 PM (Lunchtime) is the busiest hour of the day.
3.  *Revenue Driver:* Single-item orders contribute the most to total revenue.
4.  *Size Trends:* Size *L* is the dominant customer choice, while *XXL* has negligible sales.

## 🚀 Strategic Recommendations
* *Inventory:* Phase out the XXL size and focus on L and XL.
* *Marketing:* Launch "Sunday Specials" and "Late Night Deals" (after 9 PM) to boost slow periods.
* *Bundling:* Create "Combo Meals" to encourage customers to purchase more than one item per order.
